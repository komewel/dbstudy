계정--------------------------
	테이블------------------
		칼럼-------------------

PK(Primary Key, 주요한 키, 기본키)
※테이블은 무조건 PK를 가진다 PK는 NOT NULL 과 UNIQUE 의 제약 조건을 가진다.

FK(Foreign Key, 외래키) 
FK는 PK를 참조한다(참조무결성,테이블과 테이블간의 관계를 형성한다, 참조할때만 쓰인다)
FK는 1개 이상일수도(값이 1개 이상이 아니라 행이 다수 일수도 있다는거다)
       있으며 PK와는 "일(1)대 다(多)관계" 라고 부른다.
FK는 참조하는 값만 가질수 있다는게 제약조건이다.
FK는 PK가 삭제될시 FK도 삭제되어야 한다(무결성이 깨진다)

다(多) 대 다(多)관계 

상품		회원
1		A
2		B
3 		C

현실에서는 위 그림 처럼 두관계만으로도 가능하지만
DB 에서는 추가 테이블이 필요하다(총3개)
DB에서는 일대다관계가 두개가 형성되어야 다대다관계 형성된다

슈퍼키는 다수의 칼럼으로 이루어져 있지만 
후보키는 슈퍼키안에 있는 칼럼들중에 하나로만 되어있는거 관계형성이안된거
(NOT NULL, UNIQUE)를 빼온다
기본키는 사용자의 선택을 받은키라고 보면되고
대체키는 후보키중에서 기본키안에 못든 키라고 보면된다

행기반 삭제 삽입 수정이면 DML
아니다 하면 DDL

FK와 PK 관계에서 부모입장인 PK의 데이터가 삭제될시(EX.제품단종)
FK는 있어봤자이기 때문에 시스템에 오류가 생긴다 고로 두가지 방법이 있는데
1. ON DELETE CASCADE 해당PK를 참조하는 데이터를 아예 날려버린다
2. ON DELETE SET NULL 해당PK를 참조하는 데이터를 NULL 처리한다(NOT NULL로 설정되어
있는데 이걸 쓰면 안된다)

DDL, DML 차이를 알아야한다

외래키(FK)는 데이터가 새로 생성될때만 감시하지 기능을 일시중지하고 
데이터를 입력하고 커밋한 데이터는 감시하지 못한다

인덱스의 단점은 삽입 수정 삭제 할때 인덱스를 그때그때 또 추가해줘야하기 때문에 더 
오래걸린다

PK나 UNIQUE를 사용하면 빠른 이유가 고유 인덱스가 있기때문, 고로 WHERE 만들때
사용하면 좋음
